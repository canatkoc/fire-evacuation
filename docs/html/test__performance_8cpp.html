Detailed Description

Performance benchmarks for the custom data structures and simulation engine.

These tests verify structural integrity and asymptotic behavior using large inputs. Wall-clock timings are printed for information only because Valgrind slows execution. for large inputs. They do NOT use wall-clock timeouts (which are flaky in CI); instead they measure operation counts and structural integrity to confirm asymptotic behaviour — O(log n) for PriorityQueue, O(1) amortised for LinkedList ends, O(V + E log V) for Dijkstra.

Each test function follows the project convention: assert on invariants, print a "PASSED" line on success. Failures abort via assert().
