Detailed Description

Core fire-evacuation simulation engine.

FireSimulator implements two interleaved sub-tasks per simulation tick:

  • Task 3 - Dynamic Route Planner: custom Dijkstra over CellNode/Corridor topology using the project's PriorityQueue. The cost model is predictive: edge weights incorporate fire-proximity penalties and capacity congestion.
  • Task 4 - Fire Spread: per-tick propagation from ignited nodes through corridors based on per-element flammability. When a corridor's intensity crosses fire_block_threshold_ it is flagged blocked in both the internal Corridor struct and in the shared Graph.

No STL containers are used: storage is exclusively through LinkedList, Queue, PriorityQueue, Graph and intrusive pointer lists in SimEntities.h.
