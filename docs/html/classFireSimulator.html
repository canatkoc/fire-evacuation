Detailed Description

Fire-aware building evacuation simulator.

Maintains a private mirrored topology (CellNode/Corridor lists) alongside the caller-supplied Graph. Every addNode()/addCorridor() call registers the element in both structures; every fire-spread step updates blocked flags in both so that the Graph can be queried by external code (e.g., the GUI).

The simulation loop is driven by step() (one tick) or run() (multiple ticks).

Copy construction and copy assignment are disabled.

◆ findSafestExit()

EvacPath FireSimulator::findSafestExit ( const std::string &  start_id) const
inline

Finds the safest path from start_id to the nearest unblocked exit.

Runs a custom Dijkstra using the project's PriorityQueue. The edge cost function is predictiveCost(), which adds a fire-proximity penalty and a capacity penalty. Nodes at or above node_lethal_threshold_ are skipped.

Parameters
start_idID of the node from which to plan the evacuation route.
Returns
An EvacPath with reachable == true if a safe path exists.
Precondition
start_id must exist in the simulator's node list.
Postcondition
The internal Dijkstra distance table is allocated and freed locally.

◆ propagateFire()

void FireSimulator::propagateFire ( )
inline

Propagates fire through corridors and nodes for one tick.

Phase 1: each unblocked corridor absorbs fire from its endpoint nodes. Phase 2: each non-exit node accumulates fire from incoming corridors (compute-apply separated to prevent cascading within one tick).

Postcondition
fire_intensity fields updated; blocked flags set where thresholds crossed.

◆ stepPeople()

void FireSimulator::stepPeople ( )
inline

Advances all persons by one tick (three phases).

Phase A: advance persons on corridors; place arrivals in destination nodes. Phase B: drain WaitingQueues where capacity allows. Phase C: assign routes to idle persons via findSafestExit().

Postcondition
Person states updated; evacuated_count_ and casualty_count_ may increase.
